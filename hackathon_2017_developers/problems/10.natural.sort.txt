Задан набор из N строк, состоящих из цифр, маленьких латинских букв и пробелов.
Необходимо упорядочить их в неубывающем порядке естественным образом.
При естественной сортировке подстроки из цифр объединяются в десятичные числа и сравниваются арифметически.
В остальном, сравнение идет согласно кодам ASCII, в частности: пробел меньше цифры, цифра меньше буквы.

Рассмотрим примеры:
"abc00" = "abc0" - лидирующие нули значения не имеют
"abc01" = "abc1" - лидирующие нули значения не имеют
"001" = "01" - лидирующие нули значения не имеют
"789" < "1000" - сравнение чисел 
"9999" < "a" - цифра меньше буквы
"00002 abc9" < "02 abc10" - потому что "00002" = "02", " abc" = " abc", а "9" < "10"
"bc9z" < "bc10z" - различие начинается на цифре, 9 < 10
"ab00050z" < "ab00000000099z" - если убрать лидирующие нули, то получим 50 < 99
"aac55" < "abc99" - различие во втором символе, "a" < "b"
"abc def" < "abc0def" - пробел меньше цифры
"abc" < "abcd" - при равенстве префиксов меньшей считается более короткая строка

Входные данные:
Первая строка содержит числа N и K, обозначающие количество строк в наборе и номер строки в отсортированном списке, которую нужно вывести (нумерация начинается с 1).
Далее следуют N строк неупорядоченного набора.
Длина каждой строки не превосходит 500 символов.
Гарантируется, что в отсортированном списке соседние с K-й строкой строки будут отличаться от нее не только лидирующими нулями.

Выходные данные:
Найдите K-ю строку в упорядоченном по неубыванию наборе.
